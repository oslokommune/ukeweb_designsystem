If you to use Vue components or do not want to use pre-compiled CSS files in your project you can bundle and build Oslo style guide yourself for more fine grained control.

Install
Add Oslo style guide to your project using for example npm: npm install git://github.com/oslokommune/styleguide.git#1.0
Tune versioning according to semver.org.
Available versions can be found at https://github.com/oslokommune/styleguide/releases.

Build
Set up a build process using a build tool like webpack or parcel. It needs to support the sass-dart compiler and you need to handle font and svg files. Optionally vue and js depending on your needs. It is recommended to use src/osg.scss or either of the index files located in src/packages as entry point. You can of course combine a package of your own, but it requires a deeper understanding of how Oslo style guide is put together.

Other libraries
Oslo style guide is namespaced and should run alongside other namespaced libraries. However, it relies on a specific version of minireset to ensure correct behavior and sets a few global styles depending on which files you include. If other libraries set competing global styles you may end up with varying results.

Technical details on the use of maps

API-keys

The API-key that is used on this page is just for use in the styleguide, and is limited to the relevant domains.

Your project you need to provide your own API-key from Maptiler when using the Vue component or using the static map API.

If you do not change the key, you will get 403 Forbidden.


GeoJSON suported features

The map supports the following GeoJSON features:
FeatureCollection, with the following feature types: Polygon, LineString and Point.

It does not support GeometryCollection or the Multi*-variants of the feature types.


Fit map to bounds of GeoJSON-data

The map automatically sets a center point and zooms to fit the maximum bounds of the given GeoJSON-data

If a new GeoJSON dataset is provided after load, the map will automatically pan and zoom to the bounds of the new data.

This behvaiour overrides the center point and zoom level in state.

To disable this, set the property autoFitToBounds in the state to false.


Popups

Popups appear on click on a feature if the following conditions are met:

  • showPopups is true, this can be set to false by overriding the property state.
  • When using geoJson: the feature has the property description. Optionally it can also have heading.
  • When using points: description and optionally heading added to the array (per point).
  • There is actually a string with data.

Popups appear after load if the following conditions are met:

  • All of the conditions for on click are met.
  • When using geoJson: the feature has the property openPopup, with a value of true.
  • When using points: openPopup is added to the array (per point), with a value of true.
  • clusteredPoints is false.
  • Note: you should limit the number of opened popups to 1 or 2.

A link will be genereted on the heading if a url is present in the feature properties.

To display additional data after the heading and description, you can add an array called data to the feature properties.

Each item in the array must contain label and value.

This only works with the geoJson prop, and not the points prop.
